## What Was Built on March 18 (Evening Session)

### 1. Site Architecture

The prework is now a static site with config-driven theming. The architecture:

- **config.js** — A single configuration file containing institution name, instructor name/title, course title, colors, logo, mascot, welcome text, and module list. Generated by setup.html or hand-edited. Every page reads this file.
- **setup.html** — An interactive intake agent that interviews the instructor and generates config.js. Step-by-step wizard with live preview. Supports Yale, CMU, and Generic presets plus custom colors. Handles letter logos and image URL logos, mascot, welcome text.
- **index.html** — The Module 1 landing page. Config-driven banner, video placeholder with expandable transcript, roadmap, "how to use" info boxes, transition to Module 2.
- **standards-selector.html** — Interactive tool for reviewing Accounting Origins project standards and selecting which apply to the prework. Rick has completed this; output is prework-adopted-standards.md.

### 2. Video Script (v5)

The Module 1 welcome video script went through five drafts. Key decisions made through the iterations:

- **Audience:** Future MBAs, law students, policy students, future accountants and CPAs — anyone going into their first accounting course.
- **Opening move:** CVS Health complexity example, not the climate model. Climate model moved to a rabbit hole in web content.
- **Shared system emphasis:** Accounting systems are SHARED information systems that help organizations and the people who depend on them cope with complexity AND coordinate decisions and actions.
- **"Cope with the complexity of what those organizations do and what happens to them economically"** — not vague "cope with this complexity."
- **Architecture evolved, wasn't designed:** Double-entry emerged in medieval Italy. The balancing requirement crystallized through centuries of practice. Features survived because they worked.
- **Double-entry not overstated:** "Most use" not "every single one."
- **Identity is a self-imposed balancing requirement** — not something that forces balance.
- **Tax systems included** as users of double-entry and as instances.
- **Input boundary is universal:** Every system defines it. Too much data to include it all.
- **High-altitude overview framing:** "We take a high-altitude overview of that architecture. But it wouldn't be accounting if we didn't get into some specifics."
- **Debits and credits reframed:** Components of journal entry notation, which evolved for the purpose of analyzing transactions. Accountants use it to craft entries; sophisticated users use it to dig beneath the surface.
- **Prework ≠ course:** "Our goal here is to help you get into the right mindset... Think of this as coaching before the season starts."
- **Transcript:** The full v5 text lives in an expandable box below the video placeholder on the landing page, for translation and accessibility.

### 3. Navigation, Progress Bar, and Glossary

Added to index.html:

- **Sticky nav bar** at top with breadcrumb, module dropdown (8 modules, numbered dots showing visited/current/upcoming), and glossary button.
- **Progress bar** (thin accent-colored strip below nav). Tracks module visits in localStorage. Shows percentage of 8 modules visited. Each module page calls `markModuleVisited(n)`.
- **Glossary** slides in from the right. Searchable. 21 terms pre-loaded, all consistent with terminology conventions. Alphabetically sorted. Terms include: accounting system, architecture, instance, accounting identity, equities, asset, liability, double-entry, financial statements, US GAAP, IFRS, FASB, instance choice, tradeoff, compression, input boundary, journal entry notation, historical cost, balance sheet, income statement, statement of cash flows.

### 4. Standards Adopted

Rick reviewed all standards from four Accounting Origins documents and made selections via the interactive standards-selector.html. The full list is in **prework-adopted-standards.md**. Key decisions:

**Adopted as-is (23 items):** Terminology rules (no measurement, accounting identity not equation, equities plural/definitional), fundamental distinction (representations vs. reality), flag gaps, experience first, double-entry emphasis, definitions (asset/liability), prohibited behaviors, mobile responsiveness, glossary, accessibility, accounting as information system, identity as entry point, explain why behind rules, three statements connected, cash flow as reconciliation, progressive revelation, real organizations/real numbers, standards alignment phase, QA review phase, iteration process, session continuity.

**Adapted (13 items):** General language (the prework uses "we" as "you and I together"), critical perspective (tradeoffs framing instead of political analysis), structured concept delivery (first three steps apply; "hidden interests" softened to tradeoffs), color palette (configurable, not earth tones), progress bar (cross-module, not per-phase), input components, answer evaluation, dev mode, precise definitions (asset/liability apply, NFP terms don't), liquidity as asset-side question (rabbit hole for CVS), three-level hint scaffolding, planning phase, development process.

**Skipped (10 items):** Barley/measure as common standard, real dynamics/strategic concealment framing, standalone React HTML, historical liberties note, sound feedback, clay tablet display, for-profit as bridge, NFP net assets sequence, "reserves" terminology, statement of functional expenses.
